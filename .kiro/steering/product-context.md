# Product Context — SentenceFlow 제품 DNA

> 이 파일은 모든 대화에 자동 포함된다. 어떤 작업이든 여기 적힌 원칙을 **벗어나는 제안을 생성하지 않는다**.
> 상세 기획은 #[[file:docs/planning.md]] 참조.

## 정체성 (바뀌지 않는 본질)

- 프로젝트명: **SentenceFlow** (영어 문장·독해 중심 모바일 학습 앱)
- 한 줄 요약: 문장과 독해로 키우는 영어, 모르는 단어는 그 자리에서 voca(어원) + 경선식(연상)으로 해결
- 플랫폼: iOS / Android (React Native)
- 수익 모델: **무료 + 광고 + 영어 교육 제휴(CPA)**. 유료 구독은 도입하지 않는다

## 학습 원칙 (Listen-first, Speak-if-you-want)

1. 앱은 **좋은 원어민 발음을 들려주는 것과 이해를 돕는 것**까지만 책임진다
2. 따라 말하기는 **권장하되 강제하지 않는다**
3. 앱은 사용자의 발음·발화를 **평가·채점·시간 측정하지 않는다**
4. "다음" 버튼은 언제든 누를 수 있다. 발화 건너뛰기에 패널티 없음

## 기능 구조

```
[메인] 트랙 A — 짧은 문장 (회화형)     + 트랙 B — 긴 문장 (독해형)
       패턴 드릴, 단어 교체                  청킹 + 섀도잉 + 주관식 독해 퀴즈
                         │
                         │ 막힐 때마다
                         ▼
[보조] Vocab Helper (바텀 시트)
       🧬 voca(어원) + 🎨 경선식(연상) 동시 제공
       단어 단독 SRS 코스는 없음

[확장] 회화 (출시 후 단계)
       AI 파트너 롤플레이, 텍스트 교정만 (발음 평가 아님)
```

## 명시적 Non-Goals (제안 금지 항목)

다음은 제안·설계·구현 대상이 아니다. 어느 작업이든 이 항목을 부활시키면 안 된다:

- **단어 단독 학습 코스** — 단어장 메인 화면, 단어 플래시카드 SRS 등
- **발음 평가·STT 기반 점수** — 음성 인식으로 채점하는 모든 기능
- **"다 말했어요" 버튼 / 발화 타이머 / 무음 발화 시간 측정**
- **사용자 수집 기능** — 공유 시트, 클립보드 감지, 스크린샷 OCR, 내 문장 저장 등 외부 콘텐츠 반입 경로 일체
- **광고 콘텐츠의 학습 소재화** — AdMob 광고를 받아쓰기/퀴즈 재료로 가공하는 모든 시도 (근거: planning.md 9-0)
- **핵심 학습 기능의 유료화**
- **기존 교재(능률VOCA·경선식 등) 콘텐츠의 직접 복제** — 방법론만 차용, 예시는 자체 제작

## 핵심 KPI (제품 판단 기준)

| 지표 | 의미 | 목표 |
|------|------|------|
| 트랙 A/B 완료율 | 학습 성공 | A 40% / B 30% |
| 문장당 발음 재생 횟수 | 듣기 몰입도 | 평균 2회 이상 |
| 독해 퀴즈 응답률 | 주관식 참여도 | 50% |
| Vocab Helper 호출당 학습 복귀율 | 보조 모듈이 방해가 아닌지 | 80% |
| 월 광고 수익 | 지속 가능성 | 운영비 100% 커버 |

## 의사결정 톤

- 결정이 충돌할 때 **학습자 경험 > 수익 > 개발 편의** 순서로 판단한다
- 기술 선택지가 여러 개면 **무료 티어 범위 + 저비용** 을 기본으로 고른다
- 기획서에 없는 기능을 새로 제안할 때는 "왜 지금 필요한가"와 "Non-Goals에 저촉되는가"를 먼저 답한다
