# Mobile Implementation — React Native 구현 규칙

> TypeScript/JavaScript 파일이 컨텍스트에 있으면 자동 포함된다.
> Product Context의 학습 원칙을 **UI 구현 수준에서 지킨다**.

## UX 불변 규칙 (어기면 안 되는 것들)

1. **발화 평가 UI를 만들지 않는다**
   - ❌ "발음 점수", "다 말했어요" 버튼, 발화 타이머, 입 모양 애니메이션
   - ❌ 발화 건너뛰기에 대한 경고·패널티 UI
   - ✅ 원어민 발음 재생 버튼, 속도 슬라이더, 반복 재생은 항상 제공

2. **단어 단독 SRS 화면을 만들지 않는다**
   - ❌ 플래시카드 탭, 단어 학습 코스 진입
   - ✅ Vocab Helper는 **바텀 시트** 로만 존재. 단독 화면으로 승격 금지
   - ✅ "내 단어 다시 보기" 는 보조 메뉴이되 메인 탭이 되지 않는다

3. **외부 텍스트 수신 진입점을 만들지 않는다**
   - ❌ Share Extension, Intent Filter, Clipboard Listener, OCR 진입
   - 외부 수집 기능은 Non-Goal이다

4. **학습 세션 중 배너·전면 광고 금지**
   - ❌ 플레이 중 화면에 배너 띄우기, 문장 전환 사이 전면광고
   - ✅ 배너는 단어장 관리/진도 대시보드/목록 화면에만
   - ✅ 전면광고는 레슨 완료 후, 빈도 제한

5. **핵심 학습 기능을 페이월 뒤에 두지 않는다**
   - 유료 구독이 없는 앱이다. "프리미엄" 잠금 UI 자체를 만들지 않는다

## 화면 구조 원칙

| 탭 | 역할 |
|----|------|
| 홈(대시보드) | 오늘의 학습, 진도, 스트릭 |
| 트랙 A | 짧은 문장 학습 + 패턴 드릴 |
| 트랙 B | 긴 문장 학습 (청킹·섀도잉·독해 퀴즈) |
| 내 것 | 진도·설정·"내 단어 다시 보기" |

Vocab Helper는 **모달 바텀 시트** 로 어느 화면에서든 호출된다. 별도 탭 금지.

## 발음 재생 컴포넌트 요구

어떤 문장 UI에도 다음이 함께 있어야 한다:

- 🔊 재생 버튼 (크고 명확하게)
- 속도 조절 (0.5x / 0.75x / 1x / 1.25x — 긴 문장에서)
- 반복 재생 또는 구간 반복 (긴 문장에서)

재생은 **기기 TTS 폴백 + 사전 생성 오디오 캐시 우선** 로직을 따른다.

## 오프라인·동기화

- 학습 이벤트(카드 응답, 단어 탭, 독해 답안)는 **로컬 SQLite 먼저**, 성공 후 Supabase 큐로 전송
- 네트워크 복구 시 배치 동기화. 충돌은 `updated_at` 늦은 쪽 채택
- 익명 모드 지원. 로그인 시 로컬 데이터를 서버 계정에 병합

## 컴포넌트·스타일

- TypeScript strict 모드
- 함수형 컴포넌트 + Hooks
- 접근성(accessibility): 주요 버튼에 `accessibilityLabel` 필수
- 다크 모드 대응 토큰화된 컬러 사용
- 애니메이션은 `react-native-reanimated` 권장

## 데이터 모델 원칙 (클라이언트 관점)

- 엔터티는 **문장 중심**. 단어는 문장과의 관계로 표현
  - `sentences` 가 1급 테이블
  - `words` 는 참조 테이블. 사용자-단어 관계는 "미해결도" 중심
- 라이선스·출처 메타데이터를 런타임에도 유지 (향후 표시 요구 대비)

## 광고 SDK

- `react-native-google-mobile-ads` 등 RN 공식 AdMob SDK 사용
- Rewarded 완료 콜백에서만 보상 지급. 중도 종료 시 지급 금지
- 보상 유형별 일일 상한 구현

## 테스트

- 핵심 학습 루프(문장 제시 → 듣기 → 이해 확인)는 스냅샷 + 유닛 테스트
- 네트워크 의존 로직은 모킹
- Rewarded Ad 시청 → 보상 지급 경로는 통합 테스트

## 새 화면·기능을 만들기 전에

스스로에게 묻는다:

- [ ] Product Context의 Non-Goals에 저촉되지 않는가
- [ ] 발화 평가·점수 UI가 포함되지 않는가
- [ ] 학습 세션 중 광고가 끼어들지 않는가
- [ ] Vocab Helper를 단독 화면으로 승격하지 않는가
- [ ] 외부 텍스트 수신 진입점을 만들지 않는가

하나라도 "아니오"가 있으면 설계를 다시 한다.
