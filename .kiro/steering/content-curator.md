# Content Curator — 문장 콘텐츠 수집·확장·큐레이션 프로토콜

> `#content-curator` 로 명시적으로 호출해서 로드한다.
> 트랙 A(짧은 문장) / 트랙 B(긴 문장) / Vocab Helper 콘텐츠를 만들 때의 공통 프로토콜.

## 핵심 역할

**인간이 만든 씨앗에서 출발해서 LLM으로 확장하고, 다시 인간이 검증한 문장을 Supabase에 적재한다.**
0에서 LLM이 생성한 콘텐츠는 번역투·교과서투·사실 오류 리스크가 커서 메인 콘텐츠로 쓰지 않는다.

## 작업 원칙

1. **씨앗(seed)은 반드시 인간이 만든 원문에서** — Tatoeba, VOA, Simple Wiki 등
2. **LLM 확장 결과는 반드시 큐레이션을 거친다** — 자동 통과 없음
3. **라이선스 태그를 모든 문장에 부착** — 출처·라이선스·수집일 메타데이터 필수
4. **레벨(CEFR) 태그와 상황 태그를 부여** — 필터링·추천에 사용
5. **저작권 있는 외부 매체 본문은 저장·가공하지 않는다** — FOX/CNN 등은 헤드라인+링크만

## 소스별 가이드

### 트랙 A — 짧은 회화 문장

| 소스 | 라이선스 | 사용법 |
|------|---------|--------|
| Tatoeba | CC BY 2.0 FR | 메인 씨앗. 영-한 병렬 문장 중 A1~B1 빈도·길이로 필터링 |
| OpenSubtitles (OPUS) | 소스 조건부 | 구어 뉘앙스 참고용. 직접 적재는 신중 |
| VOA Everyday Grammar | 교육 목적 관대 | 대화문 섹션 발췌 |
| 미 국무부 American English | 퍼블릭 도메인 근접 | 상황 표현 보강 |

### 트랙 B — 긴 독해 문장

| 소스 | 라이선스 | 사용법 |
|------|---------|--------|
| VOA Learning English | 교육 목적 관대 | 메인 콘텐츠(초중급) |
| Simple English Wikipedia | CC BY-SA | 주제 다양성 확보. CC BY-SA는 파생물 같은 라이선스로 공개해야 하니 **재배포하지 않고 개인 학습용으로만** 사용 |
| BBC Learning English | 교육 조건부 | 중고급 보조. 약관 재확인 |
| FOX/CNN/Reuters 등 상업 매체 | — | **헤드라인 + 원문 링크만**. 본문 저장 금지 |

### Vocab Helper — voca + 경선식

| 소스 | 라이선스 | 사용법 |
|------|---------|--------|
| Etymonline | 개별 요약만 허용 | 어원 정보 씨앗. 통째 복사 금지 |
| 공개 어원 데이터셋 | CC 계열 | 구조화된 어근 정보 |
| LLM (Groq 8B) | 자체 생성 | 경선식 연상 후보 생성 — 반드시 큐레이션 |

## 제작 파이프라인

```
[1] 씨앗 수집
    소스별 파서 → 원문 + 메타데이터(출처·라이선스·수집일) → staging 테이블

[2] 1차 필터
    길이·빈도·금칙어(욕설·정치편향·고유명사 과다)로 자동 필터

[3] LLM 확장 (필요시)
    트랙 A: 상황별 변형 생성 (예: "카페에서 주문" 씨앗 → 변형 20개)
    Vocab: 어원 분해, 경선식 연상 후보 생성
    청킹: 긴 문장을 의미 단위로 분할

[4] 휴먼 큐레이션 (필수)
    체크리스트 통과 시에만 production 테이블로 승격

[5] 메타데이터 부여
    CEFR 레벨, 상황 태그, 패턴 패밀리 ID, 핵심 단어

[6] TTS 사전 생성
    자주 쓰일 문장부터 배치 생성 → Supabase Storage
```

## 큐레이션 체크리스트

문장 1개를 production으로 승격하려면 모두 통과해야 한다:

- [ ] 원어민이 실제 쓰는 자연스러운 표현인가 (번역투·교과서투 아님)
- [ ] 문법적으로 올바른가
- [ ] 타깃 레벨(CEFR)에 맞는 어휘·길이인가
- [ ] 정치·종교·성적·인종 편향 표현이 없는가
- [ ] 특정 개인·브랜드에 의존하지 않는가 (고유명사 최소화)
- [ ] 한국어 번역(제공 시)이 의미·뉘앙스를 정확히 담았는가
- [ ] 출처·라이선스 메타데이터가 부착되었는가

## Vocab Helper 전용 추가 체크

- [ ] voca: 제시된 어근·접사가 사실관계상 맞는가 (Etymonline 교차 확인)
- [ ] voca: 관련 단어 목록에 의미가 실제로 이어지는 단어만 포함
- [ ] 경선식: 한국어 연상이 원단어 발음과 충분히 겹치는가
- [ ] 경선식: 연상 문구가 성인 관점에서 민망·혐오·불쾌하지 않은가
- [ ] 두 방식 모두 학습자에게 **도움이 되는 설명인가** (단순 말장난 금지)

## Sub-agent 위임 프로토콜

`general-task-execution` 호출 시 전달 항목:

```
1. 작업 유형 (seed 수집 / LLM 확장 / 큐레이션 / 메타태깅)
2. 소스 (Tatoeba / VOA / ...)
3. 타깃 레벨 (A1 / A2 / B1 / B2 / C1)
4. 상황 또는 주제 태그
5. 수량 한도
6. 출력 포맷 (JSON 스키마 명시)
7. 이 steering 참조 지시
```

확장 결과는 **자동 적재 금지**. 반드시 큐레이션 단계를 거친 후 production에 들어간다.

## 에러·엣지 케이스

- 씨앗 수집 중 라이선스 불명 소스 발견 → staging에 넣지 말고 폐기
- LLM 확장 중 Rate Limit → 큐잉 후 재시도, 3회 실패 시 배치 중단하고 보고
- 큐레이션 거절률이 50%를 넘으면 → LLM 프롬프트 문제이니 프롬프트 튜닝 요청
- CC BY-SA 콘텐츠의 파생물은 **공개 재배포 대상이 되지 않도록** 사용자 단말 내부 학습 용도로만 제공
