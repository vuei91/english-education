# 영어 교육 앱 기획서 (초안)

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | (가칭) SentenceFlow |
| 한 줄 요약 | **문장과 독해로 키우는 영어, 필요한 단어는 그 자리에서** — 문장·독해 중심 학습 앱. 단어 암기는 어원(voca) + 연상(경선식) 방식으로 문장 학습을 뒷받침 |
| 플랫폼 | 모바일 (iOS / Android) |
| 수익 모델 | **광고 + 영어 교육 제휴 (CPA)** — 운영비 커버가 목표, 성장 시 제휴 확대 |
| 목표 출시 | MVP 3개월 내 |

## 2. 핵심 컨셉

> **"문장을 읽고 말하다 보면, 단어는 알아서 박힌다."**

**이 앱의 정체성은 "문장·독해"다.** 사용자는 **짧은 회화 문장**과 **긴 독해 문장**을 학습하며 성장하고, 앱스토어 카테고리도 독해·리딩·영어 학습이다.
단어는 **따로 외우는 대상이 아니라, 문장을 이해하기 위해 그 자리에서 해결하는 도구**다.
다만 그 도구는 **voca(어원 분해)와 경선식(발음 연상)** 이라는 **강력한 암기 방식** 으로 뒷받침된다.

### 왜 문장·독해가 메인인가

| 이유 | 설명 |
|------|------|
| **실사용 맥락** | 영어는 결국 문장 단위로 읽고 말한다. 단어만 외우면 사용으로 이어지지 않는다 |
| **의미 있는 반복** | 문장 속에서 단어를 만나면 **문맥 앵커**가 생겨 암기 강도가 올라간다 |
| **독해력 = 실무력** | 시험·뉴스·업무 메일 등 실전 영어는 대부분 "긴 문장 이해"에서 갈린다 |
| **회화로의 자연스러운 연결** | 짧은 문장 패턴 드릴이 곧 회화의 준비운동 |
| **차별점 확보** | 단어장 앱은 포화, 회화 앱은 장벽 높음, "문장·독해를 메인으로 voca·연상까지 안고 가는 앱"은 비어 있음 |

### 성장 단계 (사용자는 의식하지 않음)

```
[메인] 문장 (두 트랙)                ≫   [확장] 회화
 Sentence                                Conversation
 ────────────────                        ────────────
 🗣️ 짧은 문장 (회화형)                    충분히 쌓인 후
    패턴 드릴, 단어 교체                  AI 파트너와 롤플레이
                                         "이젠 말해볼까?" 제안
 📰 긴 문장 (독해형)
    청킹 + 섀도잉
    주관식 독해 퀴즈
    VOA·Wiki 큐레이션 콘텐츠

    │
    │ 필요할 때마다 호출되는
    ▼
[보조] 단어 해결 모듈 (Vocab Helper)
 🧬 어원 (voca 스타일)  +  🎨 연상 (경선식 스타일)
 문장에서 모르는 단어 탭 → 어원 분해 + 연상 문구 즉시 제공
 강제 암기 X. SRS는 "문장 이해도" 중심으로 돌아간다
```

핵심은 **문장·독해 경험 자체가 완결성 있어야** 한다는 점.
단어 학습은 **문장 이해를 돕는 보조 루틴**이지, 앱의 본진이 아니다.
회화는 문장이 충분히 쌓인 뒤의 **보너스이자 장기 동기부여 장치**.

## 3. 핵심 학습 원칙: 듣기 중심 + 자율 발화 (Listen-first, Speak-if-you-want)

> 문장은 **정확한 소리로 듣고, 의미를 확인**하는 것이 앱의 책임이다.
> 따라 말하기는 **권장하되 강제하지 않는다** — 평가도, 점수도, 타이머도 없다.

### 3-1. 왜 이렇게 설계하는가

| 원리 | 설명 |
|------|------|
| **정확한 입력 우선** | 좋은 발음을 충분히 들은 뒤에야 좋은 출력이 나온다. 듣기 품질이 앱의 1순위 |
| **평가 없는 자율성** | 발음 점수는 학습 동기를 해친다. 필요 없는 친구라고 느낄 때 가장 자주 돌아온다 |
| **Production Effect는 본인 선택으로** | 강제된 발화는 체감이 약하다. 자발적 발화가 기억에 훨씬 잘 박힌다 |
| **환경 독립성** | 지하철·사무실에서도 "듣기 + 마음속으로 따라 말하기" 만으로 정상 학습 가능 |

### 3-2. 발화는 "강요하지 않고 기회만 제공한다"

> 앱은 **원어민 발음을 정확하게 들려주는 것까지만** 책임진다.
> 따라 말하든 속으로 되뇌든 그냥 듣기만 하든 — **선택은 사용자의 몫**이다.

이 결정의 이유:

| 이유 | 설명 |
|------|------|
| **부담 최소화** | 평가받는다는 압박은 학습 지속성을 해친다. 듣기만 해도 충분히 돌아갈 수 있어야 한다 |
| **STT 품질 리스크 회피** | 기기 내장 STT는 한국어 억양의 영어를 자주 틀리게 인식해 오히려 좌절감을 준다 |
| **개발 비용 절감** | 발화 평가·점수·피드백 로직을 전부 걷어내면 MVP가 가벼워진다 |
| **환경 유연성** | 지하철·사무실 등 소리 내기 어려운 상황에서도 동일한 경험 |

**제공 방식**

- 문장이 표시될 때 🔊 **원어민 발음 재생 버튼**이 항상 같이 제공된다
- 재생 속도 조절(0.5x / 0.75x / 1x / 1.25x)은 **긴 문장 섀도잉 학습에 한해** 제공
- 사용자가 원하면 소리 내어 따라 말하거나, 무음으로 입모양만 움직이거나, 그냥 듣기만 해도 된다
- **앱은 사용자가 실제로 따라 말했는지 여부를 판정하지 않는다**
- "다음" 버튼은 언제든 누를 수 있다

> 발화는 학습자에게 권장되는 좋은 습관이지만 **앱이 강제하지 않는다**. Production Effect는 본인이 선택할 때 가장 크게 작동한다.

### 3-3. 학습 루프 (모든 학습의 기본 단위)

```
   ┌───────────────────────────────┐
   │ 1. 보기 (see)                  │  ← 문장 제시
   └───────────────────────────────┘
                 ↓
   ┌───────────────────────────────┐
   │ 2. 듣기 (hear) ⭐              │  ← 원어민 발음 재생 (반복 가능)
   └───────────────────────────────┘
                 ↓
   ┌───────────────────────────────┐
   │ 3. (선택) 따라 말하기          │  ← 사용자 자율. 앱은 평가하지 않음
   └───────────────────────────────┘
                 ↓
   ┌───────────────────────────────┐
   │ 4. 이해 확인 (understand)      │  ← 뜻 토글, 모르는 단어 탭
   └───────────────────────────────┘
```

이 루프는 **짧은 문장이든 긴 문장(청크 단위)이든 회화든 동일**하게 적용된다.
앱의 책임은 **좋은 원어민 발음을 들려주는 것과 이해를 돕는 것**까지이고, 발화는 사용자가 원할 때 일어난다.

### 3-4. 이 앱이 발화에 대해 하지 않는 것

- STT로 발음을 받아 점수를 매기지 않는다
- "다 말했어요" 버튼 같은 발화 시간 측정을 하지 않는다
- 발화 건너뛰기에 따른 SRS 패널티를 부과하지 않는다
- "오늘 N번 발화 완료" 같은 누적 카운트를 강요하지 않는다

대신 사용자가 자발적으로 따라 말할 확률을 높이는 데 집중한다.

### 3-5. 발화를 유도하는 수단 (강제 아님)

- 발음 재생 버튼을 **크고 명확하게** 배치
- 긴 문장 학습에서 **속도 조절 슬라이더**를 기본 제공해 따라 말하기 쉬운 속도로 튜닝 가능
- "한 번 더 듣기 / 느리게 듣기 / 구간 반복" 같은 **가벼운 보조 컨트롤** 제공
- 따라 말하고 싶은 사용자를 위해 **청크 단위 일시정지**를 선택 가능하게 (섀도잉용)

## 4. 문제 정의

- 단어 앱, 문장 앱, 회화 앱이 각각 따로 존재 → **학습 연속성이 끊김**
- 단어만 외우고 **문장·실전 사용으로 이어지지 않는** 학습 단절
- 독해 앱은 **해석 제공에 그치고** 학습자가 스스로 이해했는지 검증하지 못함
- **발화 기회 없이 눈으로만 읽어서** 문장 구조가 몸에 박히지 않음
- **회화와 독해 중 한쪽만 강한 앱** — 실무/시험 대비와 미스매치
- 대부분의 앱이 **유료 전환 압박**으로 몰입을 방해

## 5. 타겟 사용자

| 페르소나 | 특징 | 이 앱을 쓰는 이유 |
|----------|------|-----------------|
| 중급 정체자 | 단어는 아는데 문장이 안 읽히고 말이 안 나옴 | 문장 단위로 입에 붙이고 독해 감각 회복 |
| 시험 준비생 | 수능/토익/공무원 등 독해가 핵심 | 긴 문장 청킹·섀도잉 + 주관식 독해 퀴즈 |
| 실무/해외 업무자 | 영문 메일·기사·보고서 이해가 필요 | 실제 매체 문장으로 실전 독해 훈련 |
| 영어 입문 복귀자 | 다시 시작하는데 단어장은 지겨움 | 문장 중심이라 지루하지 않고, 모르는 단어는 어원·연상으로 빠르게 해결 |
| 조용한 환경 사용자 | 지하철/사무실에서 공부 | 무음 발화 모드로 부담 없이 |

## 6. 단계별 기능 설계

### 🌿 1단계: 문장 (Sentence) — **두 트랙 병행, 앱의 본진**

> 회화형 짧은 문장과 독해형 긴 문장은 **필요한 뇌 근육이 다르다.**
> 앱의 메인은 이 두 개의 병렬 트랙이다. 사용자는 자신의 목표(회화/시험/실무)에 따라 선택적으로 집중할 수 있다.

#### 🗣️ 트랙 A: 짧은 문장 (회화형)

**문장 1개 학습 플로우:**

```
Step 1. 문장 제시 + 한국어 뜻 (선택적 토글)
        "I want to eat pizza."
        → "나는 피자가 먹고 싶어."

Step 2. 🔊 원어민 발음 듣기 (반복 재생 가능)

Step 3. (선택) 소리 내어 / 속으로 따라 말하기
        → 앱은 평가하지 않음. 사용자 자율

Step 4. 모르는 단어 탭 시 즉시 Vocab Helper
        (어원 분해 + 경선식 연상 — 6-3 참조)

Step 5. 패턴 드릴로 확장 (6-1-1)
```

- 핵심 표현/생활 문장 위주
- 빈칸 채우기 / 문장 듣고 받아쓰기 등 보조 퀴즈는 **이해 확인용**으로 제공
- **문장 전체에 원어민 발음이 항상 붙어 있다** (발화 강제 없음)
- **패턴 드릴 (Substitution Drill)** — 아래 6-1-1 참조
- **2단계 회화로 자연스럽게 이어짐**

#### 📰 트랙 B: 긴 문장 (독해형)

- 뉴스·위키 등 **실제 글**에서 가져온 긴 문장 (20~40단어)
- **청킹 + 섀도잉** 중심 학습 — 아래 6-1-2 참조
- **주관식 독해 퀴즈** — 아래 6-1-3 참조
- 독해력 + 긴 문장 발화력 동시 향상
- **시험/실무 영어** 대비 가능

**잠금해제 조건**: 짧은 문장 30개 OR 긴 문장 15개 완료 시 → 2단계(회화) 오픈 (둘 중 하나만 해도 됨)

#### 6-1-1. 패턴 드릴: 문장을 몸에 심는 방법 (트랙 A)

> 같은 문장 구조를 유지한 채 **단어만 바꿔가며 반복 발화**한다.
> 시원스쿨/ALM(Audiolingual Method) 전통의 검증된 방식.
> 문장 구조가 근육에 박히고, 단어는 문맥 안에서 자동 복습된다.

**드릴 단계**

```
[Level 1] 원문 반복 듣기
  "I want to eat pizza."        🔊 × 3회
  → 원하면 따라 말하기 (자율)

[Level 2] 단어 교체 (같은 자리)
  "I want to eat sushi."        🔊
  "I want to eat a hamburger."  🔊
  "I want to eat spicy food."   🔊

[Level 3] 다른 자리 교체
  "I want to buy pizza."        🔊  (동사)
  "She wants to eat pizza."     🔊  (주어·인칭)
  "I wanted to eat pizza."      🔊  (시제)

[Level 4] 자유 응용 (슬롯 채우기)
  "I want to ____ ____."
  → 사용자가 단어를 골라 문장을 만들어 보기
  → 앱은 문장이 문법적으로 성립하는지만 간단히 확인
  → 발음은 평가하지 않음
```

**설계 포인트**

| 요소 | 원칙 |
|------|------|
| 교체 단어 선택 | **사용자가 이전 문장에서 이미 만난 단어** 우선 (연결감) |
| 한 세션 분량 | 원문 1개 + 변형 5~8개 = 약 2~3분 |
| 재생 방식 | 원어민 오디오 자동 재생 + 재생 버튼·속도 슬라이더 |
| 체크포인트 | Level 4 슬롯 채우기 문장의 **문법 성립 여부만** 간단히 확인 (발음 평가 없음) |
| 게이미피케이션 | 같은 패턴 N번 "들어보기 완료" 시 "패턴 마스터" 배지 |

**왜 이 방식이 강한가**

- **문장 구조 반복 노출** — 같은 패턴을 반복해 들으면 구조가 자연스럽게 박힌다
- **단어 복습 자동 내장** — 이전에 만난 단어가 슬롯에 재등장하며 재학습
- **자유 발화의 디딤돌** — Level 4의 슬롯 채우기가 곧 미니 회화 — 2단계(회화) 진입이 자연스러워짐
- **따라 말하고 싶으면 언제든 가능** — 발음 재생 버튼이 항상 열려 있다

#### 6-1-1-1. 트랙 A 콘텐츠 소스 전략

> 짧은 회화 문장은 **실제 인간이 만든 자연스러운 문장**에서 출발해야 한다.
> LLM으로 0에서 만들면 번역투·교과서투가 섞여 들어오기 쉽다. 씨앗(seed)은 사람, 확장은 LLM, 최종 판정은 다시 사람이 한다.

**다중 소스 + 하이브리드 생성**

| 소스 | 특징 | 라이선스 | 역할 |
|------|------|---------|------|
| **Tatoeba** | 사용자 기여 영-한 병렬 문장 DB | CC BY 2.0 FR | **메인 소스** — 일상 회화 씨앗 문장 |
| **OpenSubtitles (OPUS)** | 영어 자막 말뭉치, 구어체 비중 높음 | 소스 조건부, 학습 목적 활용 | 구어 뉘앙스 보완용 참고 |
| **VOA Everyday Grammar 대화문** | 트랙 B와 동일 소스, 짧은 대화 섹션 | 교육 목적 관대 | 레벨 A2~B1 자연스러운 대화 |
| **American English (미 국무부)** | 공공 교육 자료 | 퍼블릭 도메인 근접 | 상황별 표현 보강 |
| **LLM 확장 (Groq Llama 3.3 70B)** | 씨앗 문장 기반 변형·상황 확장 | 자체 생성 | 패턴 드릴 슬롯 치환 후보, 상황별 변형 |

**제작 파이프라인**

```
[1] 씨앗 수집
    Tatoeba·VOA에서 CEFR A1~B1 문장 약 600개 선별

[2] LLM 확장
    상황(카페·길찾기·자기소개 등 20~30개)별로
    LLM이 자연스러운 변형 20개/상황 생성 → 약 400개

[3] 휴먼 큐레이션
    번역투·어색한 표현·중복 제거
    → 최종 약 1000문장

[4] 메타데이터 부여
    CEFR 레벨 / 상황 태그 / 패턴 패밀리 ID / 핵심 단어
    → 패턴 드릴 원문 후보 100개 별도 플래그

[5] TTS 사전 생성 + 캐싱
```

**운영 원칙**

- **MVP 기준 1000문장**으로 시작 — 트랙 A 완주 가능 분량
- LLM 단독 생성은 금지. **반드시 인간 씨앗을 두고 확장**한다
- 출시 후 사용 데이터(완료율·재생 횟수)를 보며 LLM 생성 비중 점진 확대
- 성장기 콘텐츠 품질 차별화가 필요해지면 Oxford·Cambridge 상업 코퍼스 라이선스 검토

> 커리큘럼 구조는 `docs/curriculum.md` 참조.

#### 6-1-2. 청킹 + 섀도잉: 긴 문장 정복 (트랙 B)

> 긴 문장은 **의미 단위로 쪼개서** 발화하고, 원어민 음성을 **지연 따라 말하기**로 체화한다.
> 독해와 발화가 한 훈련에서 동시에 일어난다.

**4단계 학습 플로우**

```
예시 긴 문장:
"The researchers who studied the behavior of honeybees
 for over a decade finally published their findings
 in a leading scientific journal last month."

──────────────────────────────────────────
Step 1. 청킹 (의미 단위 분할)
  [The researchers]
    [who studied the behavior of honeybees]
      [for over a decade]
  [finally published their findings]
    [in a leading scientific journal]
      [last month]
  → 색깔/들여쓰기로 구조 시각화

Step 2. 청크별 듣기 + (선택) 따라 말하기
  각 청크를 하나씩 들으며 의미 확인. 원하면 따라 말해도 됨
  모르는 단어 탭 → Vocab Helper 즉시 호출 (어원 + 연상)

Step 3. 섀도잉 (Shadowing)
  원어민 음성 재생 → 0.5~1초 지연 따라 말하기
  속도 조절 (0.5x / 0.75x / 1x / 1.25x)

Step 4. 구조 요약
  "누가 / 무엇을 / 어디에 / 언제"
  → 문장 전체를 한 숨에 이해하는 감각
```

**콘텐츠 소스 전략 (다중 소스)**

| 소스 | 난이도 | 라이선스 | 역할 |
|------|--------|----------|------|
| **VOA Learning English** | A2~B1 | 교육 목적 관대 | **메인 콘텐츠** (초중급) |
| **Simple English Wikipedia** | A2~B1 | CC BY-SA | 주제 다양성 확보 |
| **BBC Learning English** | B1~C1 | 교육 조건부 | 중고급 보조 |
| **FOX/CNN/Reuters 헤드라인** | C1 | 헤드라인 + 링크만 | 고급 · 시사 맥락 |

- **기본 콘텐츠 풀**: VOA + Simple Wiki 기반으로 레벨별 100~200문장 선큐레이션
- **외부 매체**: 헤드라인 + 링크 표시, **본문 가공은 하지 않음** (약관 준수)
- **편향 방지**: 특정 매체 쏠림 없이 복수 소스 혼합

**기술적 고려**

- TTS(원어민 음성)는 **기기 TTS + 오디오 캐싱**으로 비용 최소화
- 섀도잉용 속도 조절은 **오디오 재생 속도 배율** 활용
- 청킹은 **LLM 자동 분할** (비용 이슈로 배치 처리 + 캐싱)

#### 6-1-3. 주관식 독해 퀴즈: 진짜 이해했는지 확인 (트랙 B)

> 읽고 지나가는 독해가 아닌 **"이해한 것을 자기 말로 꺼내는"** 학습.
> 수능/토익 스타일의 질문을 AI가 자동 생성하고, 자유롭게 답한 내용을 의미 기반으로 피드백한다.

**왜 주관식인가**

| 원리 | 설명 |
|------|------|
| Deep Processing | 객관식은 선택, 주관식은 **재구성** — 뇌가 더 깊이 처리 |
| Productive Output | 발화와 같은 산출 효과 — 의미를 **자기 말로** 꺼냄 |
| Comprehension Gap | 대충 안 것과 진짜 안 것의 차이가 드러남 |

**질문 유형**

| 유형 | 질문 예시 | 레벨 |
|------|---------|------|
| 주제 파악 | "이 문장이 말하고자 하는 핵심은?" | 초~중 |
| 요약 | "한국어로 한 줄 요약" | 초~중 |
| 어휘 추측 | "'bipartisan'의 의미를 문맥으로 추측" | 중 |
| 추론 | "왜 이런 상황이 벌어졌을까요?" | 중~고 |
| Paraphrase | "같은 의미를 다른 영어 문장으로" | 중~고 |
| 관점 서술 | "이 내용에 대한 당신의 생각은?" | 고 |

**피드백 방식: "정답/오답"이 아닌 "3단 피드백"**

```
사용자 답: "연구자들이 꿀벌 행동을 연구해서 저널에 실었다."

┌───────────────────────────────────────┐
│ ✅ 잘 짚은 점                          │
│  - 주체(연구자들)와 결과(저널 게재)    │
│    를 정확히 파악했어요                │
├───────────────────────────────────────┤
│ 💡 보완하면 좋을 점                    │
│  - "10년 이상" 이라는 기간 디테일이    │
│    빠졌어요                           │
│  - "finally"는 "노력 끝에" 뉘앙스      │
├───────────────────────────────────────┤
│ 📝 모범 답안 예시                      │
│  "연구자들이 10년 넘게 꿀벌 행동을     │
│   연구한 끝에 최근 유명 저널에 결과를 │
│   발표했다."                          │
├───────────────────────────────────────┤
│ [ 다시 답해보기 ] [ 다음 문장 ]        │
└───────────────────────────────────────┘
```

**레벨별 응답 언어**

| 레벨 | 답변 언어 |
|------|----------|
| 초급 (A1~A2) | 한국어 답변 허용 |
| 중급 (B1~B2) | 한국어 또는 짧은 영어 |
| 고급 (C1+) | 영어 답변 권장 |

**배치 및 운영**

- 위치: 청킹+섀도잉 완료 후 **선택적 심화 단계**
- 발화 원칙과 동일하게 **강한 권장이되 건너뛰기 허용**
- 하루 **무료 한도 5회**, 추가는 Rewarded Ad 시청으로 +3회
- Groq Llama 3.3 70B로 채점 (비용 + 속도 우위)
- 퀴즈 생성은 **배치 처리** — 사용자가 긴 문장 학습 시작 시 미리 생성

### 🌳 2단계: 회화 (Conversation)

- 이전에 학습한 **짧은 문장·패턴이 실제로 쓰이는 상황** 대화
- AI 파트너와 롤플레이 (카페, 길찾기, 자기소개)
- 자유 대화 + 문장 교정 (교정은 **문장 단위 텍스트 교정**에 한함, 발음 평가 아님)
- 대화 중 모르는 단어는 **Vocab Helper로 역참조**
- 조용한 환경 사용자는 **텍스트 입력으로도** 회화 연습 가능 — 목소리를 낼 필요 없음

### 🧰 보조 모듈: Vocab Helper — **voca + 경선식 이중 방식**

> 단어는 **따로 외우는 코스가 아니라, 문장 학습 중 막혔을 때 호출되는 도구**다.
> 다만 "도구"라고 해서 약한 것이 아니다. **voca(어원 분해)와 경선식(발음 연상)** 이라는 두 가지 강력한 암기법을 한 자리에 모은 전용 모듈이다.

#### 6-3-1. 호출 경로

단어 학습은 별도 탭이 아니라 **문장 학습 흐름 안에서 자연스럽게 끼어든다.**

| 호출 시점 | 동작 |
|----------|------|
| 짧은 문장에서 단어 탭 | 바텀 시트로 Vocab Helper 열림 |
| 긴 문장 청크에서 단어 탭 | 바텀 시트로 Vocab Helper 열림 |
| 독해 퀴즈 중 단어 탭 | 바텀 시트로 Vocab Helper 열림 |
| 회화 중 모르는 단어 탭 | 바텀 시트로 Vocab Helper 열림 |
| (선택) "내 단어 다시 보기" 메뉴 | 지금까지 탭한 단어를 한 번에 복습 |

#### 6-3-2. 🧬 voca 방식 (어원 분해, 능률VOCA 스타일)

- 접두사/어근/접미사로 단어 분해
- 예: `inspect` = `in-`(안으로) + `spect`(보다) → 조사하다
- 같은 어근 단어 그룹 노출: `respect`, `suspect`, `spectator`
- **확장성**: 어근 하나로 수십 개 단어 암기 가능
- 중급 이상 학습자에게 특히 효과적

#### 6-3-3. 🎨 경선식 방식 (발음 연상, mnemonics)

- 발음을 한국어 이미지와 연결
- 예: `abandon` → "어, 밴댕이(를) 버리다"
- **즉각적인 기억 효과**, 초급·암기 취약자에게 강점
- 짧은 스토리/이미지와 함께 제시

#### 6-3-4. UI 예시

```
문장 학습 중 "inspect" 탭
      ↓
┌──────────────────────────────┐
│  inspect   조사하다, 검사하다  │
├──────────────────────────────┤
│ [🧬 voca] [🎨 경선식] [📖 예문]│  ← 탭 전환
├──────────────────────────────┤
│  in-(안으로) + spect(보다)     │
│  → 안을 들여다보다             │
│  관련 단어: respect, suspect   │
├──────────────────────────────┤
│ [ 🔊 발음듣기 ] [ 🎤 따라말하기 ]│
└──────────────────────────────┘
```

- 기본은 **두 방식 모두 노출**, 사용자가 **선호 방식 설정** 가능
- 바텀 시트 뒷면에 voca/경선식 중 선택 또는 둘 다 표시
- "따라말하기" 를 탭하면 해당 단어를 품은 **원 문장 전체의 원어민 발음**이 재생된다 (사용자가 따라할지 여부는 자율)

#### 6-3-5. 단어 SRS는 "문장 이해도" 중심

- 단어 단독 플래시카드 코스는 **없다**
- 대신 사용자가 **문장에서 모르는 단어로 탭한 횟수·주기**를 기반으로 해당 단어의 "미해결도"를 계산
- 미해결도 높은 단어가 포함된 **문장을 우선 노출** → 단어는 문맥 안에서 자동 복습
- 원하는 사용자는 "내 단어 다시 보기" 에서 voca/경선식 카드 형태로 가볍게 복습 가능(선택 기능)

#### 6-3-6. 콘텐츠 제작 방침

- 어원: 공개 어원 데이터(Etymonline 등) 기반으로 자체 정리
- 연상: LLM으로 후보 생성 → 내부 큐레이션
- **저작권**: 기존 교재(능률VOCA, 경선식 등)의 콘텐츠를 그대로 쓰지 않고 **방법론만 차용**, 모든 예시는 자체 제작

## 7. 핵심 원칙: "연결감"

각 단계는 **독립된 기능이 아니라 하나의 흐름**이어야 한다.

| 연결 포인트 | 구현 방식 |
|------------|----------|

| 공통 UI | 동일한 캐릭터/톤/컬러 유지 |

- 광고는 **수익 채널**로만 활용한다
- 학습 콘텐츠는 **앱이 직접 제작**한다 (공개 영어 리소스 + 자체 제작)
- 광고 시청과 학습은 **UX 상 자연스럽게 연결**하되, **콘텐츠는 분리**한다
- "영어 광고 자동 학습 변환"을 다시 제안받아도 이 섹션을 근거로 **재검토하지 않는다**

### 9-1. 수익 채널 3단 구조

```
  [Tier 1] 기본 광고      →  안정적 베이스라인
   AdMob 배너/전면
           +
  [Tier 2] 보상형 광고    →  사용자 선택, 단가 높음
   Rewarded Ads
           +
  [Tier 3] 영어 교육 제휴  →  높은 매칭도, CPA
   튜터링/강의 서비스
```

### 9-2. Tier 1: AdMob 기본 광고

- **배너**: 학습 세션 사이 자연스러운 위치
- **전면 광고**: 레슨 완료 후 (빈도 제한)
- 광고 내용과 학습은 분리 — **광고는 "운영 후원"** 포지션으로 투명하게 안내
- UI 원칙: 학습 흐름을 끊지 않기, 오탭 유도 X

### 9-3. Tier 2: Rewarded Ads (보상형 광고)

- 사용자가 **선택적으로** 시청하고 보상 획득
- 보상 아이디어:
  - "하트" 회복 (오답 시 차감되는 라이프)
  - 추가 독해 퀴즈 횟수
  - 잠금된 다음 레슨 미리 열기
  - 발음 재도전 기회
- **거부감 가장 낮음**, 단가도 배너보다 높음
- MVP 단계부터 우선 적용 권장

### 9-4. Tier 3: 영어 교육 제휴 (CPA)

학습자 특성과 맞는 **영어 관련 서비스**와 직접 제휴. 일반 광고와 달리 **사용자에게도 유용한 정보**로 제시 가능.

| 카테고리 | 예시 서비스 | 제휴 방식 |
|---------|------------|----------|
| 1:1 튜터링 | 링글, Cambly, 튜터링 | 무료 체험 신청 CPA |
| 온라인 강의 | Coursera, Udemy, 야나두 | 가입/수강 CPA |
| 영어 시험 | 토익/토플/오픽 대비 서비스 | 앱 설치/가입 CPA |
| 해외 학습 | 어학연수, 교환학생 에이전시 | 상담 신청 CPA |

**제시 방식** (광고가 아닌 추천으로):
- 학습 대시보드에 "다음 단계로" 섹션 — 사용자 레벨별 맞춤 추천
- "회화 단계에 도달했어요, 실전 연습 해볼까요?" → 튜터링 서비스 추천
- 앱 내 전용 지면으로 분리, **일반 광고와 구분**

### 9-5. 단계별 수익 전략

| 단계 | 주 수익원 | 목표 |
|------|----------|------|
| MVP (0~3개월) | AdMob 배너 + Rewarded | 서버/AI 비용 커버 |
| 성장 (3~12개월) | + 영어 교육 제휴 1~2곳 | 운영비 흑자 |
| 확장 (12개월~) | + 자체 스폰서십, 프리미엄 옵션 | 지속 가능한 구조 |

### 9-6. 원칙

- **무료 유지** — 핵심 학습 기능은 절대 유료화하지 않는다
- **투명성** — 광고/제휴임을 명확히 표시
- **사용자 우선** — 수익이 학습 경험을 해치면 기능 조정
- **운영비 커버가 목표** — 수익 극대화가 아닌 지속 가능성

## 10. 기술 스택 (제안)

| 영역 | 후보 | 이유 |
|------|------|------|
| 모바일 | React Native / Flutter | 크로스 플랫폼, 빠른 개발 |
| 백엔드 | Node.js (NestJS) / Supabase | 빠른 MVP |
| DB | PostgreSQL | 학습 데이터 구조화 |
| **TTS (원어민 발음)** | **기기 TTS + 사전 생성 오디오 캐시 (자연 음성 위주)** | **앱의 1순위 경험 — 품질 좋은 음성** |
| **AI (주력)** | **Groq — Llama 3.3 70B** | **GPT-4o-mini 대비 10~20배 저렴, LPU 기반 초고속** |
| AI (경량 작업) | Groq — Llama 3.1 8B / Gemma | 단어 뜻 생성, 청킹 분할 등 반복 호출 |
| AI (폴백) | OpenAI GPT-4o-mini | Groq Rate Limit 초과 또는 품질 필요 시 |
| 인프라 | Supabase / Firebase | 무료 티어 활용 |

> **STT는 MVP에 포함하지 않는다.** 발음 평가·점수가 없기 때문에 음성 인식이 필요 없다.
> 향후 회화 단계에서 사용자 음성 입력이 필요해지면 그때 Whisper(Groq 호스팅) 등으로 도입 검토.

### 10-1. AI 스택 상세

**Groq 선택 이유**

| 장점 | 이 앱에 주는 효과 |
|------|-----------------|
| 초고속 응답 (LPU) | 독해 퀴즈/회화 피드백이 즉시 → 학습 리듬 유지 |
| 저렴한 가격 | 무료 서비스 운영비 부담 대폭 감소 |
| 무료 티어 제공 | MVP 단계부터 카드 등록 없이 사용 가능 |
| 오픈소스 모델 선택권 | 용도별 최적 모델 배정 가능 |

**용도별 모델 배정**

| 용도 | 모델 | 이유 |
|------|------|------|
| 주관식 독해 퀴즈 채점 | Llama 3.3 70B | 의미 기반 피드백 품질 |
| AI 회화 (2단계) | Llama 3.3 70B | 자연스러운 대화, 교정 품질 |
| Vocab Helper (voca/경선식 자동 생성) | Llama 3.1 8B | 대량 호출, 비용 최소화 |
| 긴 문장 자동 청킹 | Llama 3.1 8B | 단순 작업, 속도 우선 |

**리스크 & 대응**

- **Rate Limit (429 에러)**: 큐잉 + 재시도 로직 + OpenAI 폴백
- **한국어 품질**: 사전 테스트 필수, 필요 시 프롬프트 엔지니어링
- **성장기 비용**: MAU 증가 시 유료 티어 전환 또는 하이브리드 비중 조정

> 무료 서비스이므로 **운영 비용 최소화**가 중요. STT는 쓰지 않고, TTS는 자주 쓰이는 문장만 사전 생성 후 캐싱, AI 호출은 Groq 주력 + 캐싱 + 사용량 제한.

## 13. 비용 관리

- AI API 호출은 **일일 사용자당 제한** (예: 독해 퀴즈 5회/일)
- **STT 미사용** — 발음 평가가 없어 STT 서버 비용 0
- TTS는 **오디오 파일 사전 생성 + 캐싱**으로 재호출 최소화
- 청킹·Vocab Helper 생성은 **배치 처리 + 캐싱** 으로 사용자 유입 시점에 즉시 호출되지 않게
- 서버는 **Supabase 무료 티어**로 시작
- **광고 수익이 AI 비용을 넘는 시점**까지 AI 기능은 보수적으로 운영
